# Contributing to Arthais Lottery

Thank you for your interest in contributing to the **Arthais Lottery**
project.\
This document defines the standards, processes, and expectations for
contributing to any repository within the Arthais Lottery ecosystem.

By participating, you agree to follow these guidelines to ensure a
healthy, secure, and sustainable project.

Arthais Lottery is designed as a **global, decentralized, and auditable

and off-chain services.

Our priorities are:

-   Transparency
-   Security
-   Technical excellence
-   Long-term sustainability
-   Clear governance

## üß© Types of Contributions

We welcome the following types of contributions:

-   Bug reports and fixes
-   Feature proposals and implementations
-   Documentation improvements
-   Security reviews
-   Test coverage improvements
-   Infrastructure and automation enhancements

Each repository may define additional contribution scopes in its own
README.

------------------------------------------------------------------------

## üîÑ Contribution Workflow

1.  Fork the repository
2.  Create a feature or fix branch from `main`
3.  Make your changes following project standards
4.  Add or update tests when applicable
5.  Open a Pull Request with a clear description
6.  Address review feedback promptly

Pull requests that do not follow this workflow may be rejected.

------------------------------------------------------------------------

## üß™ Code Quality & Standards

All contributions must:

-   Follow the coding standards defined by the repository
-   Include tests where applicable
-   Avoid breaking backward compatibility unless explicitly discussed
-   Maintain deterministic and auditable behavior (especially for
    financial logic)

Automated checks must pass before a pull request can be merged.

------------------------------------------------------------------------

## üë• Core Contributors

Some contributors may be invited to become **Core Contributors**, which
may include:

-   Broader access to repositories
-   Participation in architectural discussions
-   Higher responsibility for code quality and security

Core Contributor status is granted at the discretion of the project
maintainers and may be revoked if expectations are not met.

------------------------------------------------------------------------

## üèõÔ∏è Maintainer Authority

Project maintainers:

-   Review and approve or reject pull requests
-   Define technical direction and priorities
-   Enforce contribution standards
-   Moderate discussions and interactions

Maintainers reserve the right to limit, suspend, or revoke contribution
privileges in cases of:

-   Code of conduct violations
-   Repeated low-quality or harmful contributions
-   Inactivity after commitment
-   Actions that put the project or users at risk
